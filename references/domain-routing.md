# Domain Routing Reference

Use this file only when the task needs domain adaptation beyond the core `SKILL.md` rules.

## Learning / Research

Use when the user wants to understand a model, library, framework, paper, API, platform, or technical concept.

Core chain:

```text
Official truth → hidden resource structure → gateway action → retained checklist
```

Expert resources often include:

- official documentation
- API reference
- specifications
- model cards or product pages
- configuration files
- source code
- official examples
- tests
- issue trackers and discussions
- maintainer comments
- reproducible notebooks or labs

Output emphasis:

- What to read first.
- What to inspect directly.
- What to run meaningfully.
- What evidence to produce.
- What to save for next time.

## Engineering / Building

Use when the user wants to build a tool, script, automation, workflow, application, integration, or agent.

Core chain:

```text
User problem → existing resources → smallest useful vertical slice → validation evidence → decision notes
```

Expert resources often include:

- existing repositories
- official SDK docs
- API limits and quotas
- official examples
- issue trackers
- architecture references
- minimal templates
- tests
- deployment docs
- pricing docs

Output emphasis:

- What not to build from scratch.
- What existing system to reuse.
- What gateway vertical slice proves value.
- What assumptions must be recorded.

## Platform Mastery

Use when the user wants to use a large platform better.

Core chain:

```text
Platform information structure → high-signal areas → noisy areas → expert workflow → gateway action → persistence outside the platform
```

Always answer:

- What is this platform best at?
- Where is primary truth located?
- Where does noise live?
- What does an expert do first?
- What gateway action exposes the real workflow?
- What should be exported into the user's own system?

## Career / Opportunity Radar

Use when the user wants to find jobs, prepare for a transition, use professional platforms, understand market demand, or build a career intelligence system.

Core chain:

```text
Opportunity sources → signal extraction → low-effort routine → profile/action updates → persistent opportunity database
```

Expert resources often include:

- company career pages
- platform-native job alerts
- hiring manager posts
- recruiter posts
- employee posts
- mutual connections
- engineering blogs
- organization activity
- newsletters
- referral network

Output emphasis:

- Treat career platforms as professional graphs, not just job boards.
- Track hiring signals, not only formal job posts.
- Convert noisy activity into structured opportunity records.
- Preserve signals outside the platform.

## Information System / Personal Knowledge Workflow

Use when the user wants to build AI news systems, RSS pipelines, research radars, job monitors, personal digests, Obsidian workflows, or knowledge bases.

Core chain:

```text
Sources → Ingestion → Normalization → AI Enrichment → Ranking → Digest → Persistence → Review
```

Expert questions:

- What decision does this system support?
- Which sources have durable signal?
- What should be filtered out?
- What schema is needed?
- What ranking criteria matter?
- Where should results persist?
- How will sources be pruned?

## Decision / Comparison

Use when the user needs to choose between tools, platforms, jobs, models, architectures, courses, products, or strategies.

Core chain:

```text
Decision criteria → evidence sources → comparison → smallest reversible test
```

Expert questions:

- What are hard constraints?
- What are preferences?
- Which evidence is primary?
- Which claims are marketing?
- What can be tested cheaply?
- What decision can be delayed?
